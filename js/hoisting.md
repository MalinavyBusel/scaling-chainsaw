Так как жс всегда сначала загружает код в память, а потом начинает исполнять, то при загрузке кода он автоматически создает в памяти места для переменных. Поэтому к ним можно обратиться ещё до того, как мы их создали. (См. [[Лексическое окружение]]) При создании лексического окружения в нем уже есть места для всех переменных, которые позже будут созданы.

Промежуток до объявления(но когда она уже создана) называется мертвой зоной. Он расположен от начала области видимости данной переменной (блок для let и const, контекст для var) и до объявления.
Если обратиться к let или const в мертвой зоне, то будет ошибка Reference Error, если к var - просто вернется undefined. Если var в strict mode, то тоже Reference Error

Функциональные выражения тоже поднимаются, но если вызвать их пока они не объявлены, то будет ошибка Type Error (пытаемся вызвать undefined как ф-ю)
var expression = function() {  
    console.log('Will this work?');  
};

Обычные ф-и тоже поднимаются и могут быть использованы сразу, а вот классы должны быть сначала объявлены, или получим ошибку
```
var Frodo = new Hobbit();  
Frodo.height = 100;  
Frodo.weight = 300;  
console.log(Frodo); **// Вывод: ReferenceError: Hobbit is not defined**  
    
class Hobbit {  
    constructor(height, weight) {  
        this.height = height;  
        this.weight = weight;  
    }  
}
```
